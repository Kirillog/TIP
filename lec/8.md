### Коэффициент

```tip
id(x) {
    return x;
}

foo(y) {
    return id(y); // c3
}

main() {
    var a, b;
    a = foo(1);  // c1
    b = foo(-1); // c2
    return a + b;
}
```
при $k = 1$ `id` вызывается из $c_3$ с аргументом $+$ (через $c_1$) и с аргументом $-$ (через $c_2$), эти потоки сливаются в $\top$. При $k = 2$ $(c_3, c_1) \neq (c_3, c_2)$, и каждый путь анализируется точно.

### Функциональный стиль

Достаточно взять powerset решетку, например из анализа живости переменных, тогда $\text{height}((Call^{\leq 2} \to P(...))^2) = O(m^2 \cdot 2^n)$, а в функциональном стиле  это $\text{height}((P(...) \to lift(P(...)))^2) = O(4^n)$, что при малом числе вызовов $m$ экспоненциально хуже для большого числа анализируемых переменных.
